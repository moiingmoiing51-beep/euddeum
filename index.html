프로젝트 폴더 구조

PD는 (복붙) yymmdd_제목 폴더를 복사해서 영상별 프로젝트 폴더로 사용합니다.

yymmdd_제목
├─ 1.mp3_to_srt
│  └─ 여기에_MP3_넣기.txt
├─ 2.csv_to_srt
│  └─ 여기에_프리미어CSV_넣기.txt
├─ 3.trans
│  └─ trans_check.xlsx
├─ 4.youtube_upload
│  └─ 업로드_결과_보관용.txt
└─ cfg
   ├─ 자막인식_키워드.txt
   ├─ 오타교정_사전.txt
   ├─ 줄바꿈금지_단어.txt
   ├─ 번역_가이드.docx
   └─ 번역예외_단어.txt
주의

BAT에서 프로젝트 폴더명 입력 시 상위 폴더명 예: 260526_STT를 입력하지 말고, 실제 프로젝트 폴더명 예: 260601_뉴발하프만 입력합니다.

1단계 · 초벌 자막 생성

입력1.mp3_to_srt/[한국어]_제목.mp3
메뉴[1] 초벌 자막 생성
출력[한국어]_제목_말자막.srt[한국어]_제목_말자막.txt[한국어]_제목_묵음보정리포트.csv

실제 첫 단어가 늦게 시작되는 경우, 자막이 묵음 구간에 너무 빨리 뜨지 않도록 start 시간이 보정됩니다.

2단계 · 자막 통합

프리미어에서 CSV로 내보낸 뒤 2.csv_to_srt 폴더에 넣고 메뉴 [2]를 실행합니다.

말자막/포인트 CSV가 있을 때

[한국어]_제목_말자막.csv[한국어]_제목_포인트.csv

시간순으로 병합하여 통합 CSV/SRT를 생성합니다.

통합 CSV가 이미 있을 때

[한국어]_제목_통합.csv

통합 CSV를 우선 사용하며, 통합 SRT만 새로 생성합니다.

생성된 [한국어]_제목_통합.csv와 [한국어]_제목_통합.srt는 3.trans로 복사됩니다.

3단계 · 다국어 번역 및 자동 검수

3.trans/trans_check.xlsx에서 번역여부가 TRUE인 언어만 번역합니다.

컬럼권장 입력설명
YT_ID유튜브 영상 ID업로드 단계에서 사용
언어영어, 일본어 등PD가 보는 언어명
코드명English, Japanese 등프로그램 내부 매칭용
번역여부TRUE/FALSETRUE인 언어만 번역
유튜브 업로드TRUE/FALSETRUE인 언어만 업로드
자동 검수

번역 후 한국어가 남아 있으면 문제 행만 자동 재번역하고 다시 검수합니다. 결과는 번역검수상태, 번역메시지에 기록됩니다.

4단계 · 유튜브 자막 업로드

유튜브 업로드=TRUE인 언어만 YouTube에 업로드합니다. 기존 자막이 있으면 새 SRT로 update 교체합니다.

  • YT_ID가 입력되어 있어야 합니다.
  • 최초 1회 브라우저 OAuth 로그인이 필요합니다.
  • 인증 후 sys/api/token.json이 생성됩니다.
  • 업로드 결과는 엑셀의 업로드상태, 업로드메시지, caption_id에 기록됩니다.
  • 상세 로그는 4.youtube_upload/upload_log.csv에 저장됩니다.

개발자용 · 아키텍처

BAT는 최소 실행만 담당하고, 한글 메뉴/프로젝트 선택은 sys/launcher.py가 담당합니다. 이 구조는 Windows CMD의 한글 인코딩 문제를 피하기 위한 설계입니다.

자막생성기.bat
└─ python "%~dp0sys\launcher.py"

sys/launcher.py
├─ mp3_to_srt.py
├─ csv_to_srt.py
├─ translate_subs.py
└─ youtube_upload_subs.py

설치 명령어

PD 또는 신규 PC에서는 아래 명령어를 PowerShell에 한 번 실행합니다.

python -m pip install --upgrade pip openai tqdm python-docx openpyxl faster-whisper rapidfuzz jamo google-api-python-client google-auth google-auth-oauthlib google-auth-httplib2

설치 확인:

python -c "from googleapiclient.discovery import build; from google_auth_oauthlib.flow import InstalledAppFlow; import openpyxl; print('OK')"

API/OAuth 관리

sys/api에는 API 설정 파일을 둡니다.

sys/api
├─ API_KEY.txt
├─ credential.json 또는 OAuth_CLIENTS.json
└─ token.json  # 최초 유튜브 인증 후 생성
보안

token.json은 인증한 유튜브 계정의 접근 권한이 들어 있으므로 배포용 압축본에서는 제외하는 것을 권장합니다.

trans_check.xlsx 처리 정책

  • 표 헤더는 1행 고정이 아니라 자동 탐색합니다.
  • 필수 헤더: 언어, 코드명, 번역여부, 유튜브 업로드
  • 상단 YT_ID 영역을 찾아 전체 언어 업로드에 적용합니다.
  • TRUE/FALSE, 1/0, 예/아니오, O/X를 bool 값으로 처리합니다.
  • trans_check를 읽지 못하면 전체 언어 번역/업로드로 fallback하지 않습니다.

문제 해결

BAT가 깨진 문자를 명령어로 실행합니다.

BAT에는 한글을 넣지 않습니다. BAT는 python "%~dp0sys\launcher.py"만 실행하고, 한글 메뉴는 Python에서 출력합니다.

trans_check.xlsx를 못 읽습니다.

openpyxl 설치 여부, 실제 헤더 행, 엑셀 파일이 열려 있는지 확인합니다. 코드에서는 헤더 행 자동 탐색을 사용해야 합니다.

No module named googleapiclient

아래 설치 명령어를 실행합니다.

python -m pip install --upgrade google-api-python-client google-auth google-auth-oauthlib google-auth-httplib2
OAuth 인증 URL이 뜹니다.

정상입니다. 최초 1회 브라우저에서 자막을 업로드할 유튜브 계정으로 인증해야 token.json이 생성됩니다.
